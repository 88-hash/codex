# 桌面端首页 /home UI 重设计任务书（Design Brief）

## 背景与目标
- 页面范围：仅 `Web 桌面端 /home`，不做手机端，不做响应式重排。
- 已读输入：`ui-audit-20260219/audit-result.json`、`ui-audit-20260219/home.desktop.png`（仅 desktop）。
- 审计关键信息：`/home` 的 `brokenImages = 15`，当前首屏与商品区存在明显视觉与可读性问题。
- 本次目标：在不改业务信息架构的前提下，将首页视觉升级为「高端、克制、清爽」（参考 Stripe / Linear / Notion 气质），同时保留“零食店”应有的轻松食欲感。

## 现状问题（Top10）
1. 商品区图片大量失效（15 张），卡片首屏大面积空白，核心卖点无法被感知。
2. 信息层级弱：Banner、功能入口、商品分区之间的主次不明确，用户扫读路径不清晰。
3. 版心与区块留白不稳定：模块间距、卡片内边距、标题与内容距离不统一。
4. 对齐精度不足：标题、操作按钮、卡片网格与分区容器的水平基线不一致。
5. 字体层级不清：正文与辅助信息字号接近，价格/标题/说明缺乏清晰的视觉等级。
6. 配色策略松散：顶部强渐变与主体浅粉背景并存，品牌色与功能色职责混乱。
7. 按钮体系不完整：主按钮与次按钮差异弱，点击意图与操作优先级不够明确。
8. 卡片质感不足：边框、阴影、圆角和状态反馈过弱，整体显得“轻飘且廉价”。
9. 内容密度失衡：卡片内容偏少但占位偏大，导致有效信息密度低、滚动成本高。
10. 品牌感不统一：当前风格偏“杂糅可爱”，与“高端克制”的体验目标不一致。

## 设计原则（桌面端）
- 仅桌面端：基准画板按 1440 宽设计，主内容区宽度建议 `1200`，整体居中。
- 栅格系统：`12` 栏栅格，栏间距建议 `24`，区块纵向节奏以 `8pt` 倍数（8/16/24/32/48）。
- 信息层级：首屏只强调一个主视觉焦点（Banner + 主行动），商品区强调“分区标题 + 卡片网格”。
- 字体层级：建议 `H1 36/44`、`H2 24/32`、`H3 18/26`、正文 `14/22`、说明 `12/18`；价格与主按钮可加粗。
- 色彩体系：中性灰做骨架（背景/边框/正文），暖橙红作为品牌主色，小面积薄荷绿或琥珀色作点缀，避免大面积高饱和混搭。
- 圆角与阴影：全站统一圆角阶梯（8/12/16）；阴影克制（低透明度、短半径），优先用边框+轻阴影体现层级。
- 组件规范：按钮、输入框、标签、卡片统一高度与状态（默认/悬停/按下/禁用），不可同页多套样式混用。
- 可读性优先：正文对比度满足可读标准，弱化装饰噪音，保证价格、标题、购买动作一眼可见。
- 保留零食店氛围：通过食欲友好的暖色点缀、少量插画/图标圆润语言体现，不走“童趣堆叠”路线。

## Pencil 提示词（可复制）
```text
请重设计「零食店」项目的 /home 首页，仅做 Web 桌面端 UI（不要做手机端，不要做响应式重排）。目标风格：高端、克制、清爽，参考 Stripe / Linear / Notion 的秩序感与留白，但保留零食店的轻松与食欲氛围。

硬性约束：
1) 仅桌面端；基准画板 1440 宽；
2) 主内容区宽度建议 1200，整体居中；
3) 仅做视觉重设计，不改业务功能与信息架构（保留：顶部导航+搜索、首屏 Banner、快捷入口、商品分区列表、页脚）。

请按以下规范输出页面：
- 布局栅格：12 栏，栏间距 24，区块纵向间距使用 8pt 体系（8/16/24/32/48）。
- 色彩建议：
  - 背景：#F7F8FA
  - 主文字：#111827，次文字：#6B7280
  - 分割线/边框：#E5E7EB
  - 品牌主色（零食氛围）：#FF6A3D（可在 #FF7A45 附近微调）
  - 点缀色（少量）：#10B981 或 #F59E0B
  - 禁止大面积高饱和渐变覆盖主内容区，只允许在 Banner 或局部装饰轻量使用。
- 字体层级：
  - H1: 36/44, Semibold
  - H2: 24/32, Semibold
  - H3: 18/26, Medium
  - Body: 14/22, Regular
  - Caption: 12/18, Regular
  - 价格与主要 CTA 允许更高字重。
- 按钮规范：
  - 主按钮：实色填充（品牌主色），白字，圆角 10，高度 40；
  - 次按钮：描边或浅底，文字深灰；
  - 提供默认/悬停/按下态，主次层级一眼可辨。
- 卡片与列表：
  - 商品卡片采用统一圆角（12）+ 1px 边框 + 轻阴影；
  - 图片区比例统一，避免大面积空白占位；
  - 标题最多 2 行，价格与购买按钮视觉优先级明确；
  - 卡片间距统一，首屏可读密度提升。
- Header 规则：
  - 高度统一（建议 72 左右），左侧品牌与导航，中间搜索，右侧账户/动作；
  - 减少杂色，使用中性背景或极浅品牌色，保证导航可读性。
- Footer 规则：
  - 分区清晰（品牌、导航、帮助、联系），字色对比达标；
  - 风格与整体一致，避免过重压暗或信息拥挤。
- 视觉气质：
  - 更克制、更整洁、更有秩序；
  - 强化对齐与留白，不要可爱化堆砌；
  - 保留“零食店”温度，但整体高级感优先。

请最终输出一版可直接用于桌面端落地的首页视觉方案。
```

## 设计验收标准
1. 页面仅体现桌面端方案，主内容区约 1200 且居中，未出现移动端布局语义。
2. 主要模块（Header、Banner、分区标题、商品网格、Footer）在同一栅格体系下对齐一致。
3. 文本层级清晰：标题、正文、说明、价格区分明确，长时间阅读不费力。
4. 色彩克制：中性色占主导，品牌色用于关键动作与强调，不出现高饱和噪音。
5. 按钮层级明确：主次操作在 3 秒内可被识别，交互状态完整。
6. 卡片密度合理：单位屏可见有效商品信息明显增加，不再出现“空白大于信息”。
7. 阴影与圆角统一：组件视觉语言一致，无同屏多套风格冲突。
8. 可读性达标：正文与背景、次级文字与背景均有足够对比度。
9. 品牌氛围准确：既有零食类目的亲和度，又具备现代电商的高级克制感。
10. 整体观感较现状明显提升：对齐更准、节奏更稳、视觉更干净。
